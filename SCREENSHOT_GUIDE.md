# App Store Screenshot Guide

## Overview
Screenshots are crucial for App Store conversion. They're often the first thing users see and can make or break a download decision.

## Required Sizes

### iPhone (Required)
- **6.7" Display (iPhone 14 Pro Max, 15 Pro Max):** 1290 x 2796 pixels
- **5.5" Display (iPhone 8 Plus):** 1242 x 2208 pixels

### iPad (Required) 
- **12.9" Display (iPad Pro):** 2048 x 2732 pixels
- **11" Display (iPad Pro):** 1668 x 2388 pixels

## Screenshot Creation Process

### 1. Prepare Sample Data
Before taking screenshots:
- Create 5-6 realistic sample jobs
- Add client information (use fake but realistic names)
- Attach sample photos to jobs
- Set various statuses and due dates
- Add pricing information

### 2. Device Setup
- Use a clean iOS device or simulator
- Ensure good lighting if using physical device
- Clean screen of fingerprints
- Close other apps to avoid notifications
- Set device to Do Not Disturb mode

### 3. Capture Screenshots
- Use iOS Screenshot feature (Volume Up + Power button)
- Take multiple shots of each screen
- Ensure UI elements are fully loaded
- Check for any loading states or errors

### 4. Edit Screenshots
**Tools:** Canva, Figma, Photoshop, or Screenshot Creator apps

**Editing Checklist:**
- Add title overlays with readable fonts
- Ensure consistent branding colors
- Remove any personal information
- Add subtle shadows or frames if desired
- Optimize for App Store display

### 5. Title Overlay Design
**Font:** San Francisco (iOS system font) or similar
**Size:** Large, bold, readable
**Color:** White text with dark background or dark text with light background
**Position:** Top third of image
**Background:** Semi-transparent overlay for readability

## Quality Checklist

### Before Submitting:
- [ ] All required sizes captured
- [ ] No personal/real client information visible
- [ ] Text is readable on all devices
- [ ] Screenshots show key app features
- [ ] Consistent branding and colors
- [ ] No UI glitches or loading states
- [ ] Professional sample content
- [ ] Title overlays are clear and compelling

### Technical Requirements:
- [ ] RGB color space
- [ ] 72 DPI resolution
- [ ] No transparency
- [ ] JPEG or PNG format
- [ ] Under 8MB file size each

## Pro Tips

### Content Strategy:
1. **Show, Don't Tell:** Let the interface speak for itself
2. **Use Real Scenarios:** Create believable photography jobs
3. **Highlight Benefits:** Each screenshot should show a key benefit
4. **Progressive Story:** Screenshots should flow logically
5. **Professional Polish:** Everything should look production-ready

### Design Tips:
1. **Consistent Branding:** Use your green color throughout
2. **Clean Interface:** Remove clutter, focus on key features
3. **Readable Text:** Ensure all text is legible at small sizes
4. **Visual Hierarchy:** Guide the eye to important elements
5. **Professional Photos:** Use high-quality sample images

### Common Mistakes to Avoid:
- Using Lorem Ipsum or placeholder text
- Showing empty states or loading screens
- Including personal information
- Inconsistent data between screenshots
- Poor image quality or resolution
- Cluttered or confusing layouts

## Final Review

Before uploading to App Store Connect:
1. View screenshots on different devices
2. Check readability at thumbnail size
3. Ensure story flows logically
4. Verify no sensitive information is visible
5. Confirm all technical requirements met

Remember: Great screenshots can significantly increase your download rate. Take time to make them perfect!
